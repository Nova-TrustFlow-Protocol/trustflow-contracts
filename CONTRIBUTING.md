# Contributing to TrustFlow Contracts

Thank you for contributing to TrustFlow. Because we handle B2B financial transactions, code quality and security are paramount.

## Development Workflow
1. **Fork & Clone:** Fork the repository.
2. **Branching:** Use `feat/`, `fix/`, or `chore/` prefixes.
3. **Testing:** You MUST write unit tests in `src/test.rs` covering all state transitions for your logic.
4. **Formatting:** Ensure `make fmt` passes before submitting a Pull Request.

Please search the codebase for `// TODO:` comments to find areas for immediate improvement!
